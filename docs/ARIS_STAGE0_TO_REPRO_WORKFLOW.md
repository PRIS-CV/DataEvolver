# ARIS Stage0 To Reproduction Workflow

This document describes the upstream entry that connects Stage0 websearch to
paper reproduction and dataset construction inside ARIS.

## What Stage0 Does

Stage0 turns a research task into a small, high-signal paper shortlist plus a
structured summary.

Formal handoff outputs:

- `ARXIV_LINKS_AND_SUMMARY.md`
- `arxiv_links_and_summary.json`
- `NEXT_AI_REPRO_PROMPT.md`

`research_prior.json` is also produced, but it is advisory only.

## Flow

```text
research task
-> Stage0 websearch / arXiv candidate filtering
-> ARXIV_LINKS_AND_SUMMARY.md / arxiv_links_and_summary.json
-> NEXT_AI_REPRO_PROMPT.md
-> downstream agent follows AI_REPRODUCTION_GUIDE.md
-> method-line decomposition
-> small-scale / real-small / synthetic proxy reproduction
-> dataset construction
-> quality gating
-> REPRO_STATUS.json / artifact_lineage.json / final report
```

## Entry Points

Primary CLI:

```bash
python scripts/stage0_web_research.py init --query "<research task>" --output-dir runtime/research_priors/<batch>/<task_slug> --tag <tag>
python scripts/stage0_web_research.py add-evidence ...
python scripts/stage0_web_research.py finalize --session-dir runtime/research_priors/<batch>/<task_slug> --max-papers 5
python scripts/stage0_web_research.py validate --prior-path runtime/research_priors/<batch>/<task_slug>/research_prior.json
```

Compatibility alias:

```bash
python scripts/run_ldr_stage0_research.py prepare ...
```

## What The Downstream Agent Must Do

`NEXT_AI_REPRO_PROMPT.md` points the downstream agent to:

- read the paper links and summary
- choose the best candidate or candidates
- read `runtime/AI_REPRODUCTION_GUIDE.md`
- create `docs/METHOD_CONSTRAINT_CARD.md`
- track output provenance
- keep the reproduction level explicit
- update `REPRO_STATUS.json` as work progresses

## Provenance Labels

- `official_downloaded`: downloaded from paper author / official repo / official data source
- `official_result_recomputed`: recomputed from official artifacts
- `self_generated`: generated by the current pipeline
- `mock_or_proxy`: simplified stand-in for interface / structure testing
- `synthetic_proxy`: generated surrogate for unavailable real assets
- `not_completed`: required but not yet reproduced

## Reproduction Level

Use one of these labels at the end of the downstream run:

- evaluation smoke reproduction
- small-scale method reproduction
- real-small method reproduction
- synthetic proxy branch
- full paper-level reproduction

## Download / Cost Rules

The downstream agent must stop and ask for confirmation when a step requires:

- a large download
- a closed model
- a paid API
- extra credentials
- a task likely to exceed the user's compute or time budget
- a task expected to run longer than the user allowed window
