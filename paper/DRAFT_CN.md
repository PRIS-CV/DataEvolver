# ARIS：基于自由文本 VLM 反馈的合成数据自动构建方法——面向视角可控的图像编辑

**技术报告**

---

## 摘要

我们提出 **ARIS**（**A**utonomous **R**endering with **I**terative **S**cene-aware feedback），一个面向图像编辑任务的全自动合成数据构建流水线。构建高质量的 3D 感知图像编辑配对训练数据——例如视角可控的旋转编辑——传统上需要在 Blender 等工具中对渲染参数进行大量人工调试，这一过程既耗时又难以规模化。ARIS 通过引入*基于自由文本 VLM 引导的渲染进化循环*来解决这一问题：视觉语言模型（VLM）以自然语言对渲染场景进行审阅，AI agent 直接读取该反馈，从结构化动作空间中选择下一步渲染调整动作，迭代提升视觉质量，直到审阅者明确确认结果可接受为止。

我们将该流水线实例化，构建了 **ARIS-Rotate** 数据集：包含 50 个物体、350 组旋转编辑配对，每个样本将一张标准正面视角图像与一个自然语言指定的目标视角配对（例如 *"将该物体从正面视角旋转到右侧视角"*）。消融实验表明，VLM 进化循环相比单次渲染（无迭代优化）显著提升了渲染质量。使用 ARIS-Rotate 数据对 Qwen Image Edit 2511 进行 LoRA 微调后，模型在 PSNR、SSIM 和 LPIPS 指标上均优于基础模型，验证了自动构建数据的下游训练价值。

---

## 1. 引言

高质量的配对训练数据是监督式图像编辑模型的基础。对于 3D 感知的编辑任务——如将物体旋转到指定视角、改变光照条件或替换材质——每个训练样本需要一张*源图像*、一张不同视角或条件下的*目标图像*，以及一条对应的*编辑指令*。从真实场景采集此类数据成本极高且费时费力，而现有的合成方法则面临关键的质量瓶颈。

**合成数据流水线的质量鸿沟。** 一种标准做法是自动渲染 3D 资产来生成配对图像，成本较低。然而，朴素渲染经常产出带有瑕疵的场景：平淡的光照与场景环境不匹配、物体与背景之间的色彩不一致、物理上不合理的物体放置（悬浮或穿入地面）以及阴影质量差。现有流水线通常通过刚性评分规则或基于数值阈值选择渲染调整的固定控制器来进行质量控制——这些方法缺乏诊断和解决复杂视觉问题所需的语义理解能力。

**我们的方法。** 我们提出 ARIS，一个通过*基于自由文本 VLM 引导的进化循环*来提升渲染质量的流水线。我们不依赖结构化评分或固定规则，而是使用视觉语言模型（VLM）以自然语言审阅渲染场景，识别具体的视觉问题，如平淡光照、色偏或接地不足。AI agent 直接读取这些自由文本反馈，从离散渲染动作空间中选择动作来解决诊断出的问题——调整主光强度、环境旋转、材质参数或物体位置。该过程反复进行，直到 VLM 审阅者明确确认结果可接受。

AI agent 替代了人类美术师在将 3D 资产集成到 Blender 场景时通常需要进行的手动试错过程：一旦流水线验证通过，大规模数据集构建便不再需要逐个物体的专家调参，从而降低成本并提升一致性。

**实例化：视角可控的旋转编辑。** 我们将 ARIS 实例化到一个具体且可评估的任务上：构建*旋转条件图像编辑*数据集，模型需要将物体从标准正面视角旋转到自然语言指定的目标视角（如 *"右侧视角"*、*"左后方视角"*）。我们构建了 **ARIS-Rotate**，包含 50 个物体 × 8 个水平视角，产生 350 组编辑训练对。

**实验结果。** 消融实验表明，VLM 进化循环相比无迭代优化的单次渲染，显著提升了渲染质量。使用 ARIS-Rotate 对 Qwen Image Edit 2511 进行 LoRA 微调后，模型在 PSNR、SSIM 和 LPIPS 上均优于基础模型，证实了自动构建数据的下游训练价值。将优化后数据训练的 LoRA 与未优化数据训练的 LoRA 进行对比，进一步证明了进化循环所产出数据的更高实用性。

**贡献：**

1. **基于自由文本 VLM 引导的渲染进化循环。** 我们引入了一种质量优化循环：AI agent 读取 VLM 审阅者的自由文本反馈，据此选择渲染动作，以语义感知的自适应方式替代了刚性的基于评分的控制器。

2. **场景感知 Blender 渲染集成。** 我们描述了一种保留真实场景光照和环境的渲染系统，将 3D 物体集成到已有的 Blender 场景文件中，同时通过 24 个结构化原子动作支持逐物体的光照、放置、材质和场景参数调整。

3. **ARIS-Rotate 数据集与验证。** 我们构建了一个 50 物体、350 对的旋转编辑数据集，使用自然语言视角提示，并通过对 Qwen Image Edit 2511 的 LoRA 微调验证了其训练价值。

## 2. 相关工作

**面向图像编辑的合成数据。** 大量工作探索了为指令跟随式图像编辑自动构建配对训练数据。InstructPix2Pix [待引用] 结合 GPT-4 [待引用] 生成编辑指令和 Stable Diffusion [待引用] 渲染源/目标图像对来生成训练数据。MagicBrush [待引用] 引入人工标注的真实编辑配对来弥补纯合成数据的分布偏差。Emu Edit [待引用] 使用多样化的多任务数据集扩展指令跟随编辑能力。这些方法聚焦于 2D 语义编辑任务；构建*几何可控*的配对——特别是多视角旋转——需要显式的 3D 建模，这是 2D 扩散模型流水线无法提供的。ARIS 通过完整的 3D 到渲染流水线及迭代质量控制填补了这一空白。

**3D 感知数据生成。** Objaverse [待引用] 及其后续版本提供了大规模 3D 资产库，用于渲染合成多视角数据。Zero-1-to-3 [待引用]、SyncDreamer [待引用] 和 Wonder3D [待引用] 等方法在此类数据上训练新视角合成模型。这些工作聚焦于几何重建和视角合成，但未解决场景感知渲染质量问题或产出适用于指令跟随模型的*训练就绪*编辑配对所需的迭代质量控制。ARIS 通过将 3D 重建与场景感知 Blender 渲染和基于 VLM 的质量进化相结合来弥合这一差距。

**VLM 作为反馈与评判者。** 使用语言模型评估或改进其他系统输出的做法已获得广泛关注。LLM-as-a-Judge [待引用] 等工作表明，强大的 LLM 可以作为人类评估的代理。在多模态场景中，近期工作探索了 VLM 用于图像质量评估 [待引用] 和偏好对齐 [待引用]。我们的方法不同之处在于，VLM 反馈不是用于*事后评估*，而是作为迭代渲染循环中的*主动信号*：AI agent 读取审阅者的自由文本，直接决定调整哪些渲染参数，在评估与生成之间形成闭环。

## 3. 方法

### 3.1 流水线总览

ARIS 将自然语言物体描述转化为一组高质量渲染图像，包含六个阶段。该流水线完全自动化：从初始概念描述到最终质量审批的渲染结果，无需任何人工介入。

> **[图 1 占位]** 横向流程图：文本扩展 → T2I 生成 → SAM3 分割 → 3D 重建 → 场景感知渲染 → VLM 审阅循环

**阶段 1：文本扩展。** 给定一个种子概念（如 *"木椅"*），语言模型将其扩展为详细的 T2I 提示词，包含材质、颜色、几何结构和表面纹理描述。该步骤提升了提示词的具体性，改善下游生成图像的质量和多样性。

**阶段 2：物体图像生成。** T2I 模型（Qwen-Image-2512）使用扩展后的提示词生成 1024×1024 的物体参考图像，背景干净。

**阶段 2.5：前景分割。** SAM3 对物体前景进行分割，输出透明背景的 RGBA 图像。这一步移除了偶然的背景内容，同时高保真地保留物体。关键的是，SAM3 仅对物体区域操作，因此后续 3D 重建（阶段 3）不会包含地板平面等背景几何体，消除了对后处理网格清理的需求。

**阶段 3：单图 3D 重建。** Hunyuan3D-2.1 使用前馈式单图重建模型，从 RGBA 图像重建带纹理的 3D 网格（GLB 格式）。生成的网格包含基于物理的渲染（PBR）纹理贴图，支持在多种光照条件下进行逼真渲染。

**阶段 4：场景感知 Blender 渲染。** 重建的网格被插入到真实 Blender 场景中，使用 Cycles 路径追踪进行渲染。详见 3.2 节。

**阶段 5.5–5.6：VLM 审阅与渲染进化。** VLM（Qwen3.5-35B-A3B）以自由文本形式审阅渲染图像。AI agent 读取审阅内容，选择渲染参数调整来解决已识别的质量问题。该循环重复进行，直到审阅者明确确认结果可接受。详见 3.3 节。

---

### 3.2 场景感知 Blender 渲染

渲染每个 3D 物体需要将其合理地放置在场景中，匹配场景光照环境，并从一致的视角进行渲染。我们使用一个固定的 Blender 场景文件（一个已布置的室内场景）作为背景环境。

**场景环境保留。** 一个关键设计原则是*不*覆盖场景已有的光照：世界环境（HDRI）和已有灯光从场景文件中保留。主灯光按固定系数（0.3）缩放而非禁用，确保物体接收自然的场景照明。这避免了在与场景背景冲突的人造工作室光照下渲染物体的常见失败模式。

**物体放置。** 物体网格通过射线检测（raycast）程序放置在场景地面上：系统从物体包围盒向下发射射线，检测真实地面表面高度，确保物理合理性。物体被定位为其底部位于场景地面上，并可通过可配置的垂直偏移进行微调。

**渲染配置。** 所有渲染使用 Cycles 路径追踪，512 个采样，1024×1024 分辨率，提供高质量的全局光照和阴影细节。对每个物体，我们渲染八个水平视角，方位角分别为 0°、45°、90°、135°、180°、225°、270° 和 315°，对应第 4 节定义的自然语言视角标签。

### 3.3 基于自由文本 VLM 引导的渲染进化

ARIS 的核心贡献是通过 VLM 反馈驱动渲染图像迭代优化至可接受视觉质量的进化循环。

> **[图 2 占位]** 左列：Round 0 渲染 → Round 1 → ... → 最终（keep）。右列：每轮 VLM 自由文本摘要 + agent 选择的动作。

**VLM 审阅。** 每轮渲染后，VLM（Qwen3.5-35B-A3B，启用扩展思考模式）审阅渲染图像，产出自由文本形式的自然语言评估。审阅者被提示识别具体的视觉问题，包括：

- `flat_low_contrast` — 平淡或低对比度光照
- `color_shift` — 物体与场景之间的色彩不一致
- `scene_light_mismatch` — 光照方向与场景不匹配
- `shadow_missing` — 接触阴影缺失或较弱
- `floating` / `ground_intersection` — 物理上不合理的放置
- `scale_implausible` — 物体相对场景过大或过小
- `material_too_plastic` — 材质过于光滑或不真实

审阅者以三种结论之一结束：**keep**（结果可接受）、**revise**（建议具体修改）或 **reject**（存在根本性问题需要重新渲染）。

**AI agent 决策。** AI agent 从跟踪记录中读取完整的自由文本审阅内容，识别主要诊断问题。基于诊断结果，agent 从渲染动作空间（表 1）中选择一个或多个动作。这种设计不同于刚性的基于评分的控制器：agent 解读的是视觉故障的语义描述，而非机械地响应数值阈值。

**动作空间。** 渲染动作空间包含 24 个原子动作，组织为四个组：

**表 1：渲染动作空间**

| 组别 | 示例动作 | 数量 | 目标参数 |
|------|---------|------|---------|
| 光照 | `L_KEY_UP`、`L_KEY_YAW_POS_15` | 4 | 主光强度、主光偏转角 |
| 物体 | `O_LIFT_SMALL`、`O_SCALE_UP_10` | 6 | 垂直偏移、偏转旋转、缩放 |
| 场景 | `ENV_ROTATE_30`、`ENV_STRENGTH_UP` | 5 | HDRI 偏转、环境强度、接触阴影 |
| 材质 | `M_VALUE_DOWN`、`M_ROUGHNESS_UP` | 9 | 饱和度、明度、色相偏移、粗糙度、高光 |
| **合计** | | **24** | |

**问题到动作的映射。** Agent 使用从诊断问题到候选动作的结构化映射。例如：
- `flat_low_contrast` → `ENV_STRENGTH_UP`
- `color_shift` → `M_VALUE_DOWN_STRONG`、`M_SATURATION_DOWN`
- `scene_light_mismatch` → `ENV_ROTATE_30`、`L_KEY_YAW_POS_15`
- `shadow_missing` → `S_CONTACT_SHADOW_UP`、`L_KEY_UP`
- `floating_visible` → `O_LOWER_SMALL`

**防震荡与步长衰减。** 为防止 agent 在对立动作之间反复震荡，系统跟踪每个参数调整方向的历史记录。如果某参数的方向已翻转三次或以上，该参数将在后续优化中被冻结。动作步长随轮次衰减（第 0 轮：100%，第 1 轮：70%，第 2 轮及之后：50%），以在最优值附近实现细粒度收敛。

**终止条件。** 循环在 VLM 审阅者明确产出 *keep* 结论时终止，或在最多五轮后终止。只有获得明确 keep 结论的结果才被视为质量通过。

## 4. ARIS-Rotate 数据集

### 4.1 构建

我们应用 ARIS 流水线构建了 **ARIS-Rotate**，一个面向旋转条件图像编辑的数据集。ARIS-Rotate 中的每个样本将物体的标准正面视角图像与目标视角图像和自然语言编辑指令配对。

**物体选择与多样性。** 我们生成了 50 个物体，涵盖家具、日用品、工具、装饰品和户外设备等多种类别。物体描述覆盖了多种材质（木材、金属、塑料、织物）、形状（紧凑型、细长型、对称、非对称）和色彩搭配。

**视角定义。** 每个物体在八个等间距水平方位角下进行渲染。我们使用自然语言标签而非数值角度来表示视角，反映用户自然描述旋转指令的方式：

| 方位角 | 视角标签 |
|--------|---------|
| 0° | front view（正面视角） |
| 45° | front-right view（右前方视角） |
| 90° | right side view（右侧视角） |
| 135° | back-right view（右后方视角） |
| 180° | back view（背面视角） |
| 225° | back-left view（左后方视角） |
| 270° | left side view（左侧视角） |
| 315° | front-left view（左前方视角） |

**编辑配对与指令模板。** 我们将 0°（front view）的渲染结果指定为标准源图像。对其余七个目标视角，使用以下指令模板构建编辑配对：

> `Rotate this object from front view to {target_view}.`

每个物体贡献 7 组编辑配对，共计 **350 组训练对**。完整渲染图像集包含 **400 张图像**（50 个物体 × 8 个视角）。

> **[图 3 占位]** 网格：行 = 4 个代表性物体，列 = 8 个视角。每个单元格展示经 VLM 进化后该视角的渲染图像。

### 4.2 数据划分

我们使用**物体不相交（object-disjoint）**划分方式：同一物体不能同时出现在训练集和测试集中。这确保了评估衡量的是对未见物体的泛化能力，而非对已见物体不同视角的记忆。

| 划分 | 物体数 | 配对数 | 图像数 |
|------|--------|--------|--------|
| 训练 | 35 | 245 | 280 |
| 验证 | 5 | 35 | 40 |
| 测试 | 10 | 70 | 80 |
| **合计** | **50** | **350** | **400** |

## 6. 局限性

**动作空间覆盖。** 当前 24 个离散动作空间对某些失败模式的覆盖有限。特别是 `flat_low_contrast`（场景整体色调范围低）和 `color_shift`（物体色调偏离场景）并非总能通过材质和光照调整完全解决。部分物体经过 40 轮以上的进化仍未达到 keep 标准，表明这些失败模式需要当前动作空间之外的能力——例如场景背景替换或全局色调映射调整。

**数据集规模与类别覆盖。** ARIS-Rotate 包含 50 个物体。虽然足以证明流水线的可行性并验证下游效用，但该规模不足以支撑一个通用旋转编辑模型所需的广度。类别覆盖也有限；某些特殊或非典型物体类型可能超出本次验证集的分布范围。

**仅旋转验证。** 我们仅在旋转这一单一编辑任务上验证了 ARIS。该流水线被设计为可泛化到其他几何可控编辑任务（光照变化、材质替换、尺度变化），但此类泛化性尚未在本报告中得到实验验证。

**VLM 审阅者噪声。** Qwen3.5-35B-A3B 审阅者每次查询的评分方差约为 0.006–0.012，可能对边界案例产出不一致的结论。重复审阅或多视角聚合可以缓解这一问题，但审阅者可靠性仍是流水线质量的一个变量。

**未公开发布。** 当前实现依赖于特定的 Blender 场景文件和服务器环境，未随本技术报告公开发布。

## 7. 结论

我们提出了 ARIS，一个面向图像编辑任务的自动化合成数据集构建流水线。核心贡献是基于自由文本 VLM 引导的渲染进化循环：AI agent 读取 VLM 审阅者的自然语言反馈，从结构化动作空间中选择渲染参数调整，迭代至审阅者确认质量可接受。这一方法以自动化反馈循环替代了人工 Blender 参数调试，实现了可扩展且一致的 3D 数据集构建。

我们将 ARIS 实例化到旋转条件图像编辑任务，产出 ARIS-Rotate：一个 50 物体、350 对的数据集，使用自然语言视角指令。消融实验证实了进化循环相比单次渲染对渲染质量的提升。在 ARIS-Rotate 上对 Qwen Image Edit 2511 进行 LoRA 微调，在测试集上展现了相比基础模型的可度量改善，验证了自动构建数据的下游效用。

**未来工作。** 若干方向可自然延伸本工作。首先，扩展动作空间以覆盖当前未解决的失败模式（全局色调映射、背景替换）将提升覆盖率。其次，将 ARIS 应用于更多编辑任务——光照变化、材质替换、尺度变化——将验证方法的通用性。第三，扩大物体规模（500+）并拓宽类别覆盖，将为训练更强的编辑模型提供所需的数据量。最后，集成自适应轮次预算，为更难的物体分配更多进化轮次，将提升大规模运行时的效率。
