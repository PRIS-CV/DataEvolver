**DataEvolver — 基于 VLM 引导的自动合成数据构建系统**    *2026.03 – 2026.05*

**Situation（背景）：** 3D 感知的图像编辑任务（如视角可控旋转编辑）需要大量高质量的配对训练数据。传统方法依赖人工在 Blender 中逐物体调参（光照、材质、放置），耗时且难以规模化；现有自动化流水线采用固定评分规则进行质量控制，缺乏语义理解能力，无法诊断和修复复杂渲染瑕疵。

**Task（任务）：** 设计并实现一套端到端全自动合成数据构建流水线，用 VLM（视觉语言模型）的自由文本反馈替代人工质检，通过 AI Agent 驱动的迭代优化闭环自动提升渲染质量，最终产出可用于模型微调的训练数据集。

**Action（行动）：**

- 设计了六阶段自动化流水线：种子概念 → LLM 文本扩展 → T2I 图像生成（Qwen-Image-2512）→ SAM3 前景分割 → Hunyuan3D-2.1 单图 3D 重建 → 场景感知 Blender 渲染，实现从自然语言描述到高质量渲染图像的全链路自动化
- 核心创新：构建了 **VLM 引导的渲染进化循环**——VLM（Qwen3.5-35B-A3B）以自由文本形式审阅渲染场景并诊断具体视觉问题（平淡光照、色偏、悬浮等），AI Agent 读取反馈后从 24 个结构化原子动作（覆盖光照/物体/场景/材质四组参数）中选择调整策略，迭代优化直至质量通过
- 设计了防震荡机制（方向翻转追踪 + 参数冻结）和步长衰减策略（100% → 70% → 50%），确保优化收敛
- 构建了双层反馈闭环：内循环（VLM 质检门控过滤低质量资产）+ 外循环（评测指标 → 弱角度识别 → 定向补充数据 → 重新训练），实现数据集的自演化
- 实例化为 **DataEvolver-Rotate** 数据集：50 个物体 × 8 个水平视角，350 组旋转编辑配对，采用物体不相交划分（35 train / 5 val / 10 test）
- 对 Qwen Image Edit 2511 进行 LoRA 微调验证数据集的下游训练价值；项目开源至 GitHub 并构建了项目展示页面

**Result（成果）：**

- 消融实验证实 VLM 进化循环相比单次渲染显著提升了渲染质量；LoRA 微调后在 SpatialEdit-Bench（488 对）上 PSNR 从 14.83 提升至 **16.79**，LPIPS 从 0.3033 降至 **0.2478**，CLIP-I 从 0.8823 提升至 **0.9525**
- 项目开源至 GitHub（[PRIS-CV/DataEvolver](https://github.com/PRIS-CV/DataEvolver)），配套项目页面：[DataEvolver](https://pris-cv.github.io/DataEvolver/)
- 撰写了完整技术报告，涵盖方法设计、消融实验与定量验证

**DataEvolver：VLM引导的自演化合成数据构建系统 | 数据算法实习生 2026.03 – 2026.05**

中国电信人工智能研究院（TeleAI） / 天翼AI

项目背景：基于 VLM + AI Agent 驱动的自演化数据构建框架，通过自由文本反馈替代固定评分规则，实现从种子概念到训练就绪数据集的全链路自动化闭环。

- **VLM-Guided 渲染进化循环**：由于传统 Blender 合成数据依赖人工逐物体调参，我们设计 24 原子动作空间 + VLM 自由文本反馈的 Agent 决策闭环，自动诊断并修复光照、材质、场景等渲染瑕疵，替代人工质检。
- **双层反馈自演化架构**：鉴于单轮数据构建无法针对性提升模型弱项，我们设计内循环（VLM 质量门控过滤）+ 外循环（评测反馈 → 弱角度定向补数据 → 重训练）的双层架构，实现数据集持续自我进化。
- **防震荡收敛控制**：针对 Agent 参数优化中方向反复反转问题，设计方向翻转追踪 + 参数冻结与步长衰减调度（100%→70%→50%），确保渲染质量稳定收敛。
- **落地成果**：项目开源至 [GitHub](https://github.com/PRIS-CV/DataEvolver)，技术报告已上传至 arXiv，同时构建了[项目展示页面](https://pris-cv.github.io/DataEvolver/)；LoRA 微调后在 SpatialEdit-Bench 上 PSNR 16.79（+1.96），CLIP-I 0.9525，全面超越基线。
